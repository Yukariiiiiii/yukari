从零搭建个人博客系统

2026-06-20 968 阅读 15 分钟
全栈 Node.js 部署

从一行代码到线上部署，完整走一遍个人博客的搭建流程。不依赖 WordPress、不买服务器，用纯静态 + 免费托管实现你的个人写作空间。

架构选型

个人博客有三种主流方案，各有取舍：

方案优点缺点适合人群
纯静态 (HTML+CSS)最快、最便宜、最安全无后台、手写 HTML极简主义者
静态生成器 (Hexo/Hugo)Markdown 写作、主题丰富需构建步骤技术博主
动态博客 (WordPress/Ghost)功能完整、有后台需服务器、维护成本内容创作者

本文选择 纯静态 + SPA 模式，原因：

  • 零成本托管（GitHub Pages / Vercel 免费）
  • 加载速度极致（无后端延迟）
  • 完全可控（CSS/JS 随便改）
  • 配合 Git 就是版本管理

SPA 路由核心

用 History API 实现无刷新跳转，音乐播放器不中断：

JavaScript
// 拦截所有站内链接
document.addEventListener('click', async e => {
    const link = e.target.closest('a[data-link]');
    if (!link) return;
    e.preventDefault();
    const url = link.href;

    // fetch 拉取新页面
    const html = await fetch(url).then(r => r.text());
    const doc = new DOMParser().parseFromString(html, 'text/html');

    // 只替换 <main> 内容
    document.querySelector('main').innerHTML = doc.querySelector('main').innerHTML;

    // 更新 URL（不刷新）
    history.pushState({}, '', url);
});
关键：<audio> 和播放器放在 <main> 外面，SPA 切换时永远不会被销毁，音乐自然不断。

写作工作流

用 Markdown 写文章，构建时转为 HTML 插入模板：

Node.js 构建脚本
// build.js
const fs = require('fs');
const { marked } = require('marked');
const template = fs.readFileSync('template.html', 'utf-8');

fs.readdirSync('posts').forEach(file => {
    const md = fs.readFileSync(`posts/${file}`, 'utf-8');
    const html = marked(md);
    const title = md.match(/^#\s+(.+)$/m)?.[1] || 'Untitled';
    const output = template
        .replace('{{title}}', title)
        .replace('{{content}}', html);
    fs.writeFileSync(`articles/${file.replace('.md', '.html')}`, output);
});

部署上线

方案 A：GitHub Pages（免费）

Bash
git init
git add .
git commit -m "first blog"
git remote add origin https://github.com/你的用户名/你的用户名.github.io
git push -u origin main
# 在 GitHub Settings → Pages → Source 选 main 分支

方案 B：Vercel（自动部署 + HTTPS）

  1. 把项目推到 GitHub 仓库
  2. 登录 vercel.com → Import Project
  3. 选仓库 → 点 Deploy
  4. 每次 git push 自动更新
