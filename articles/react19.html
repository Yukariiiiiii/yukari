React 19 新特性详解

2026-06-28 3.1k 阅读 8 分钟
React Hooks RSC

React 19 带来了近年来最大的 API 变革：Actions、use() Hook、Server Components 正式落地、forwardRef 被废除。本文逐一拆解每个变化的用法和迁移策略。

Actions：告别 useState + useEffect 表单地狱

Actions 是 React 19 最核心的新概念。它把"用户操作 → 服务端处理 → 状态更新"这条链路由框架统一管理。

React 19
// 之前：手动管理 pending / error / success
const [isPending, setIsPending] = useState(false);
const [error, setError] = useState(null);

async function handleSubmit(e) {
    e.preventDefault();
    setIsPending(true);
    try {
        await saveData(formData);
    } catch (e) {
        setError(e);
    } finally {
        setIsPending(false);
    }
}

// React 19：useActionState
const [state, formAction, isPending] = useActionState(
    async (prev, formData) => {
        return await saveData(formData);
    },
    null
);
核心收益：pending 状态、乐观更新、错误处理全部由框架管理，组件代码减少 40%。

use() Hook：异步数据的新范式

不再需要 useEffect + useState 的 boilerplate，直接在组件内 use() 一个 Promise 或 Context：

JavaScript
import { use } from 'react';

function Comments({ commentsPromise }) {
    // use() 会暂停组件直到 Promise resolve
    const comments = use(commentsPromise);
    return comments.map(c => 
{c.text}
);
}

// 也适用于 Context
const theme = use(ThemeContext);

Server Components 正式稳定

RSC 不再是实验特性。组件文件顶部加 'use server' 或 'use client' 即可声明运行环境。

指令运行环境能做什么
'use server'Node.js / Edge数据库查询、文件系统、密钥操作
'use client'浏览器事件处理、useState、useEffect
无指令两端皆可纯渲染组件

forwardRef 废除

React 19 中 forwardRef 不再需要——ref 可以直接作为普通 prop 传递：

React 19
// 之前
const Input = forwardRef((props, ref) => (
    <input ref={ref} {...props} />
));

// React 19：直接传
function Input({ ref, ...props }) {
    return <input ref={ref} {...props} />;
}

// 使用方式完全不变
<Input ref={myRef} />

useOptimistic：乐观更新的官方方案

JavaScript
function LikeButton({ postId, likes }) {
    const [optimisticLikes, addOptimisticLike] = useOptimistic(
        likes,
        (state) => state + 1
    );

    async function like() {
        addOptimisticLike(); // 立即 +1
        await fetch(`/api/like/${postId}`, { method: 'POST' });
    }

    return <button onClick={like}>❤️ {optimisticLikes}</button>;
}

迁移清单

  • ✅ 升级 React 19 + ReactDOM 19
  • ✅ 移除所有 forwardRef → 改为 ref prop
  • ✅ 移除 React.Fragment 简写以外的用法统一
  • ✅ 用 useActionState 重构表单
  • ✅ 用 useOptimistic 替代手动乐观更新
  • ⬜ 逐步引入 Server Components
