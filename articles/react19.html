React 19 新特性详解

2026-06-28 3.1k 阅读 约 15 分钟
React 前端 新特性

React 19 带来了什么？

React 19 是近年来最大的一次更新，核心围绕三个方向：Server Components 稳定化、Actions 简化表单、底层性能优化。它不是"加了一堆 API"，而是重新定义了 React 应用的架构范式。

一句话总结：React 19 = Server Components 正式可用 + 数据变更变得像写同步代码一样简单 + 底层重写带来更快的渲染。

一、Server Components（RSC）

Server Components 允许你在服务器上渲染组件，把结果以 RSC 流式协议发送到客户端。它解决了传统 SSR 的痛点：

对比项传统 SSRServer Components
JS 包大小所有组件都打包到客户端Server 组件零 JS 发送到客户端
数据获取客户端 useEffect 里 fetch直接在组件里 async/await
SEO✅ 好✅ 更好（流式 HTML）
交互性hydration 后才能交互Client 组件保持交互

注意：Server Component 里不能使用 useState、useEffect、浏览器 API。需要交互的部分用 'use client' 标记的子组件。

二、Actions：表单提交的终极简化

以前：满屏 useState + fetch + loading 状态

function Form() {
    const [name, setName] = useState('');
    const [loading, setLoading] = useState(false);
    const [error, setError] = useState(null);

    async function handleSubmit(e) {
        e.preventDefault();
        setLoading(true);
        try {
            await fetch('/api/submit', {
                method: 'POST',
                body: JSON.stringify({ name })
            });
        } catch (err) {
            setError(err.message);
        } finally {
            setLoading(false);
        }
    }
    // ... 一堆 JSX
}

现在：useActionState + 服务端函数

'use server';
async function submitForm(prevState, formData) {
    const name = formData.get('name');
    // 直接操作数据库
    await db.user.create({ data: { name } });
    return { ok: true };
}

'use client';
import { useActionState } from 'react';

function Form() {
    const [state, formAction, isPending] = useActionState(submitForm, null);
    return (
        <form action={formAction}>
            <input name="name" />
            <button disabled={isPending}>提交</button>
            {state?.ok && <p>成功！</p>}
        </form>
    );
}

核心变化：

  • action 属性直接接收服务端函数
  • useActionState 自动管理 pending / error / data
  • 乐观更新用 useOptimistic
  • 不需要手写 fetch、loading 状态

use() 的特点：

  • 可以在条件分支和循环里调用（不像 Hook 有规则限制）
  • 配合 Suspense 自动处理 loading
  • 同时支持 Promise 和 Context

四、底层性能优化

1. 编译器优化（React Compiler）

React 19 配套推出了React Compiler（独立包），自动给组件加 memo：

// 以前要手动 memo
const ExpensiveItem = memo(function ExpensiveItem({ data }) {
    return 
{heavyCompute(data)}
;
});

// React Compiler 自动分析依赖，自动 memoize
function ExpensiveItem({ data }) {
    return 
{heavyCompute(data)}
;
}

2. 更细粒度的更新调度

19 重写了调度器，支持优先级中断——用户输入永远优先于数据渲染：

import { useTransition } from 'react';

function SearchResults() {
    const [isPending, startTransition] = useTransition();
    const [query, setQuery] = useState('');

    function onChange(e) {
        const value = e.target.value;
        startTransition(() => {
            setQuery(value); // 低优先级更新
        });
    }
    // ...
}

五、破坏性变更（迁移注意）

  • react-dom/client 的 render 已移除，必须用 createRoot
  • propTypes 完全移除
  • 事件池（SyntheticEvent pooling）移除
  • 默认开启 useFormStatus、useOptimistic
  • 需要 Node 18+ 和现代浏览器
迁移建议：先用 react@19 + react-dom@19 升级依赖，再逐步将表单改为 Action 模式，最后引入 React Compiler。不要一次性全改。

小结

React 19 的本质是把数据获取和变更写回组件内部，借助 Server Components 和 Actions 消除样板代码。配合 Compiler 自动优化，开发者可以专注于业务逻辑而非性能调优。这是 React 向"全栈组件"迈出的最大一步。
