JavaScript 性能优化指南

2026-07-12 856 阅读 12 分钟
JavaScript 性能优化 事件循环

前端性能优化是一个系统工程，而 JavaScript 往往是最大的瓶颈所在。本文从事件循环、内存管理、渲染优化三个维度，帮你建立完整的性能优化思维框架。

理解事件循环

JavaScript 是单线程的，所有异步操作都依赖事件循环（Event Loop）调度。理解它是性能优化的第一步。

JavaScript
// 宏任务 vs 微任务执行顺序
console.log('1');              // 同步
setTimeout(() => console.log('2'), 0); // 宏任务
Promise.resolve().then(() => console.log('3')); // 微任务
console.log('4');              // 同步
// 输出: 1 → 4 → 3 → 2
关键点：微任务（Promise / queueMicrotask）会在当前宏任务结束后立即执行，优先于下一个宏任务（setTimeout / setInterval）。

避免阻塞主线程

长时间运行的 JS 会阻塞渲染，导致页面卡顿。解决方案：

1. 使用 Web Worker

JavaScript
// 主线程
const worker = new Worker('heavy-task.js');
worker.postMessage(largeData);
worker.onmessage = (e) => {
    console.log('计算结果:', e.data);
};

// heavy-task.js
self.onmessage = (e) => {
    const result = e.data.map(item => expensiveCalc(item));
    self.postMessage(result);
};

2. 任务分片

JavaScript
// 将大循环拆分为多个微任务
async function processInChunks(items, chunkSize = 100) {
    for (let i = 0; i < items.length; i += chunkSize) {
        const chunk = items.slice(i, i + chunkSize);
        chunk.forEach(processItem);
        // 让出主线程
        await new Promise(r => setTimeout(r, 0));
    }
}

内存管理

常见内存泄漏场景

场景原因解决方案
未清理的事件监听元素移除后监听仍在removeEventListener / AbortController
定时器未清除setInterval 持续引用组件销毁时 clearInterval
闭包引用变量被闭包持有无法 GC及时置 null 断开引用
DOM 引用缓存已移除元素仍在数组中使用 WeakMap / WeakSet

使用 WeakMap 避免内存泄漏

JavaScript
// WeakMap 的 key 是弱引用，DOM 移除后可被 GC
const elementData = new WeakMap();

elementData.set(domElement, { clicks: 0, lastSeen: Date.now() });
// 当 domElement 从 DOM 树移除且无其他引用时，数据自动被回收

渲染性能

避免强制同步布局

读写交替会导致浏览器强制重新计算布局：

JavaScript
// ❌ 错误：读 → 写 → 读 → 写（触发多次 reflow）
items.forEach(item => {
    const h = item.offsetHeight; // 读
    item.style.height = h * 2 + 'px'; // 写
});

// ✅ 正确：先批量读，再批量写
const heights = items.map(item => item.offsetHeight);
items.forEach((item, i) => {
    item.style.height = heights[i] * 2 + 'px';
});

使用 transform 代替 top/left

transform 走 GPU 合成层，不会触发 layout 和 paint：

CSS
/* ❌ 触发 layout */
.element { top: 100px; left: 50px; }

/* ✅ 仅触发 composite */
.element { transform: translate(50px, 100px); }

工具推荐

  • Chrome DevTools Performance：录制和分析运行时性能
  • Lighthouse：自动化性能评分
  • web-vitals：在代码中监控 Core Web Vitals
  • requestIdleCallback：在空闲时执行非关键任务
JavaScript
// 空闲时执行低优先级任务
requestIdleCallback((deadline) => {
    while (deadline.timeRemaining() > 0 && tasks.length > 0) {
        processTask(tasks.pop());
    }
}, { timeout: 2000 });

总结

性能优化的核心原则：

  • 🔍 先测量再优化：用数据说话，不要盲目优化
  • 🧵 不阻塞主线程：长任务拆分 / Web Worker
  • 🧹 及时释放资源：清理监听、定时器、引用
  • 🎨 减少重排重绘：批量读写、用 transform
  • ⏳ 延迟非关键任务：IntersectionObserver / requestIdleCallback
