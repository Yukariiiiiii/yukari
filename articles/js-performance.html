JavaScript 性能优化指南

2026-07-12 856 阅读 约 12 分钟
JavaScript 性能 前端

为什么要关注 JS 性能？

JavaScript 是单线程语言，任何长时间运行的同步代码都会阻塞主线程，导致页面卡顿、交互无响应、动画掉帧。用户感知到的"流畅"通常要求主线程单次任务不超过 16ms（对应 60fps）。

核心原则：让主线程保持空闲。把重活儿拆小、延后、或丢给别的线程（Web Worker）。

一、理解事件循环

JS 运行时靠事件循环（Event Loop）调度任务。理解它是优化的前提：

调用栈 (Call Stack)
   ↓ 空闲时
从 微任务队列 (Microtask Queue) 取任务 [Promise.then / queueMicrotask]
   ↓ 清空微任务后
从 宏任务队列 (Macrotask Queue) 取一个任务 [setTimeout / setInterval / I/O]
   ↓ 渲染帧（如需要）
回到循环开头

关键结论：

  • Promise.then 回调是微任务，会在当前同步代码执行完后立即执行
  • setTimeout(fn, 0) 是宏任务，至少等一帧
  • 长时间占用调用栈 = 阻塞渲染 = 页面卡死

二、减少主线程阻塞

1. 分解长任务

如果一个函数要循环处理 10 万条数据，别一口气跑完：

// ❌ 阻塞主线程
function processAll(items) {
    items.forEach(item => heavyCompute(item));
}

// ✅ 用 requestIdleCallback 分片
function processChunk(items, start = 0) {
    const CHUNK = 1000;
    const end = Math.min(start + CHUNK, items.length);
    for (let i = start; i < end; i++) heavyCompute(items[i]);
    if (end < items.length) {
        requestIdleCallback(() => processChunk(items, end));
    }
}

2. 使用 Web Worker

纯计算任务（排序、加密、压缩）丢给 Worker，完全不占主线程：

// main.js
const worker = new Worker('worker.js');
worker.postMessage(bigData);
worker.onmessage = e => console.log('结果:', e.data);

// worker.js
self.onmessage = e => {
    const result = e.data.sort((a, b) => a - b);
    self.postMessage(result);
};

3. 防抖与节流

高频事件（scroll、resize、input）必须限制触发频率：

// 防抖：等用户停下来再执行
function debounce(fn, delay) {
    let t;
    return (...args) => {
        clearTimeout(t);
        t = setTimeout(() => fn(...args), delay);
    };
}

// 节流：每隔固定时间执行一次
function throttle(fn, interval) {
    let last = 0;
    return (...args) => {
        const now = Date.now();
        if (now - last >= interval) {
            last = now;
            fn(...args);
        }
    };
}

三、内存管理

常见内存泄漏场景

  • 遗忘的定时器：setInterval 不清除 → 闭包引用永远存活
  • detached DOM：移除节点后仍被 JS 变量引用
  • 事件监听未解绑：SPA 路由切换后旧组件监听仍在
  • 全局变量累积：不断往 window 上挂东西
// ✅ 组件销毁时清理
class Component {
    constructor() {
        this.onResize = this.handleResize.bind(this);
        window.addEventListener('resize', this.onResize);
        this.timer = setInterval(() => this.tick(), 1000);
    }
    destroy() {
        window.removeEventListener('resize', this.onResize);
        clearInterval(this.timer);
    }
}

四、渲染性能

1. 避免强制同步布局（Layout Thrashing）

交替读写会触发浏览器反复重排：

// ❌ 每次循环都触发重排
for (let i = 0; i < els.length; i++) {
    const w = els[i].offsetWidth;  // 读
    els[i].style.width = w * 2 + 'px'; // 写
}

// ✅ 先读完再写
const widths = els.map(el => el.offsetWidth);
els.forEach((el, i) => el.style.width = widths[i] * 2 + 'px');

2. 使用 transform 和 opacity

这两个属性走 GPU 合成层，不触发重排/重绘：

/* ❌ 触发重排 */
.element { left: 100px; top: 50px; }

/* ✅ GPU 加速 */
.element {
    transform: translate(100px, 50px);
    will-change: transform;
}

五、加载性能

  • <script defer>：等 DOM 解析完再执行，不阻塞渲染
  • <script type="module">：天然 defer + 严格模式
  • 代码分割（Code Splitting）：路由级懒加载
  • Tree Shaking：只打包实际用到的导出
  • 预加载关键资源：<link rel="preload">
测量优先于优化：用 Chrome DevTools 的 Performance 面板录制真实场景，找到瓶颈再动手。盲目优化往往适得其反。

小结

JS 性能优化的本质就是三件事：少占主线程、少碰 DOM、少加载代码。掌握事件循环、善用工具测量、遵循"测量→分析→优化→再测量"的循环，你的页面就能丝般顺滑。
