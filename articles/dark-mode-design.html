深色模式设计的最佳实践

2026-07-05 2.3k 阅读 约 10 分钟
设计 CSS 用户体验

为什么深色模式不只是"反色"

很多人以为深色模式就是背景变黑、文字变白。如果真这么做，效果往往是刺眼、廉价、不可读。好的深色模式需要考虑：亮度层级、色彩饱和度、对比度、材质质感。

核心原则：深色模式不是"黑底白字"，而是用低亮度色板 + 微妙的光影层次营造深度感。

一、色彩系统

不要用纯黑

纯黑（#000）在屏幕上看起来像"洞"，缺乏层次。主流产品的深色背景都在 #121212 ~ #1e1e2e 之间：

产品深色背景
Material Design#121212
Apple iOS#000000（但叠半透明层提亮）
VS Code Dark+#1e1e1e
GitHub Dark#0d1117
本博客#0f0e17

建立亮度层级

至少定义 3 层背景色，用亮度差表达"前后关系"：

:root[data-theme="dark"] {
    --bg-base:   #0f0e17;  /* 最底层：页面背景 */
    --bg-elev-1: #1a1925;  /* 一级浮起：卡片 */
    --bg-elev-2: #252335;  /* 二级浮起：弹窗、下拉 */
    --text-high:  #e8e8f0;  /* 主要文字 */
    --text-mid:   #a0a0b8;  /* 次要文字 */
    --text-low:   #6c6c80;  /* 辅助文字 */
}

二、对比度与可读性

WCAG 对比度标准

  • AA 级：正文 ≥ 4.5:1，大字号 ≥ 3:1
  • AAA 级：正文 ≥ 7:1，大字号 ≥ 4.5:1

常见错误：用高饱和蓝（#2196F3）直接做深色背景上的文字，对比度只有 ~3:1，不达标。

检测工具：Chrome DevTools → Rendering → Emulate CSS media → prefers-color-scheme: dark，配合 WebAIM Contrast Checker 验证。

避免"光晕效应"

深色背景上的浅色文字如果太亮（#FFFFFF），周围会出现光晕，看起来模糊。建议正文用 #E0E0E0 ~ #E8E8F0 而非纯白。

三、用 CSS 变量实现一键切换

/* 默认深色 */
:root { --bg: #0f0e17; --text: #e8e8f0; }

/* 浅色覆盖 */
:root[data-theme="light"] {
    --bg: #f0f0f5;
    --text: #1a1a2e;
}

/* JS 切换 */
document.documentElement.setAttribute('data-theme', 'light');

进阶：尊重用户系统偏好：

@media (prefers-color-scheme: dark) {
    :root:not([data-theme]) {
        --bg: #0f0e17;
        --text: #e8e8f0;
    }
}

四、图片与媒体的适配

1. 降低亮色图片的刺眼感

@media (prefers-color-scheme: dark) {
    img, video {
        filter: brightness(0.85) contrast(1.1);
    }
}

2. 使用 CSS Mask 适配图标

单色图标用 mask-image 替代背景图，可以直接用 color 控制颜色：

.icon {
    width: 24px; height: 24px;
    background: currentColor;
    mask-image: url('icon.svg');
    -webkit-mask-image: url('icon.svg');
}

五、毛玻璃 + 深色 = 绝配

本博客就是典型案例——深色半透明 + backdrop-filter 模糊，既保持暗色基调，又有玻璃般的质感层次：

.glass {
    background: rgba(255, 255, 255, 0.05);  /* 微微透出背后色彩 */
    backdrop-filter: blur(16px) saturate(180%);
    border: 1px solid rgba(255, 255, 255, 0.12);
}

六、常见踩坑

  • 阴影消失：深色背景上 box-shadow: 0 2px 8px #000 几乎看不见，改用 rgba(0,0,0,0.5) + 提亮边框
  • 焦点环不可见：:focus-visible 的 outline 要在深色下重新配色
  • 滚动条刺眼：自定义 ::-webkit-scrollbar 配色
  • 模态框无层次：用半透明遮罩 rgba(0,0,0,0.6) + 模糊背景

小结

深色模式的核心是建立亮度层级 + 控制对比度 + 尊重用户偏好。不要简单反色，要用 CSS 变量构建完整的色彩系统，让深浅模式切换如丝般顺滑。本博客的切换体验就是这套方案的实战成果。
