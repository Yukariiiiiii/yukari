深色模式设计的最佳实践

2026-07-05 2.3k 阅读 6 分钟
设计 深色模式 CSS变量

深色模式不只是把白色背景换成黑色。糟糕的深色模式会让眼睛更累、对比度刺眼、色彩失真。本文总结一套经过验证的深色模式设计原则，帮你做出真正舒适的暗色体验。

为什么不是 #000？

纯黑（#000000）在 OLED 屏幕上虽然省电，但它会带来严重问题：

  • 光晕效应：白色文字在纯黑背景上会产生"光晕"，长时间阅读眼睛疲劳
  • 层次感缺失：纯黑无法再"加深"来表达层级
  • 色彩过饱和：在纯黑上，彩色会显得过于鲜艳刺眼

推荐使用 #121212（Material Design）或 #1a1a2e（本博客）作为基础深色：

CSS
:root[data-theme="dark"] {
    --bg: #1a1a2e;        /* 不是纯黑 */
    --text: #e8e8f0;       /* 不是纯白 */
    --text-muted: #6c6c80;  /* 降低对比度 */
    --border: rgba(255,255,255,0.08);
}

对比度原则

WCAG 建议正文对比度至少 4.5:1，大号文字至少 3:1。但深色模式下有个陷阱：

不要这样做：用纯白文字（#FFFFFF）配纯黑背景（#000000），对比度 21:1 反而刺眼。建议正文用 85%~90% 白色，如 #E0E0E0。
元素推荐亮度说明
背景8%~15%深色但不纯黑
表面/卡片12%~18%比背景略亮一层
正文文字85%~90%清晰但柔和
次要文字55%~65%用于辅助信息
强调色饱和度降低 10%避免深色底上过亮

层级表达

深色模式中，表达层级的正确方式是加亮而非加暗：

CSS
/* 层级从低到高，亮度递增 */
--surface-0: #121212;  /* 页面背景 */
--surface-1: #1e1e2e;  /* 卡片 */
--surface-2: #2a2a3e;  /* 悬浮卡片 */
--surface-3: #363650;  /* 弹出层/对话框 */

/* 用边框增加层次 */
border: 1px solid rgba(255,255,255,0.06);

色彩适配

浅色模式下的品牌色直接搬到深色模式往往太亮。需要调整：

  • 降低饱和度：深色底上高饱和色会"发光"
  • 提高亮度：让色彩在暗底上仍然可见
  • 避免大面积纯色：用渐变或半透明替代
CSS
:root[data-theme="light"] {
    --accent: #6c4cf0;   /* 浅色底上的品牌紫 */
}
:root[data-theme="dark"] {
    --accent: #9b7fff;   /* 深色底上更亮、略降低饱和 */
}

技术实现

CSS 变量 + data-theme 切换

CSS
/* 定义两套变量 */
:root[data-theme="dark"] { --bg: #1a1a2e; --text: #e8e8f0; }
:root[data-theme="light"] { --bg: #f0f0f5; --text: #1a1a2e; }

/* 全局使用变量 */
body { background: var(--bg); color: var(--text); }

/* JS 切换 */
document.documentElement.setAttribute('data-theme', 'dark');

尊重用户系统偏好

CSS
/* 默认跟随系统 */
@media (prefers-color-scheme: dark) {
    :root { --bg: #1a1a2e; --text: #e8e8f0; }
}
@media (prefers-color-scheme: light) {
    :root { --bg: #f0f0f5; --text: #1a1a2e; }
}

避坑清单

✅ 该做：

  • 用 CSS 变量统一管理颜色
  • 阴影改用半透明亮色（深色底上暗阴影看不见）
  • 图片加暗色滤镜或混合模式适配
  • 测试 OLED 上的效果（纯黑区域真的熄屏）

❌ 别做：

  • 直接 invert() 整个页面（色彩会失真）
  • 用纯黑 #000 做背景
  • 用纯白 #FFF 做正文
  • 忽略图片/视频的亮度适配
